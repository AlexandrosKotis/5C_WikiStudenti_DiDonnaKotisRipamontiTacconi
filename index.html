Progetto Wiki

Si vuole realizzare una web application per la consultazione di una wiki, accessibile sia ad utenti senza account sia con permessi speciali. L'obbiettivo del progetto è fornire una piattaforma collaborativa in cui le voci delle wiki possano essere create, modificate e approvate in maniera controllata.

  • Classe/Anno: 5°C Informatica - 24/25
  • Author: Angelo Ripamonti, Alexandros Kotis, Lorenzo Di Donna e Simone Tacconi
  • Creato: 26 Febbraio 2025
  • Ultima modifica: 10 Marzo 2025

Requisiti Funzionali

La piattaforma sarà strutturata in tre principali interfacce utente:

  1. Interfaccia per moderatori/amministratori:
    • Approva o respinge le modifiche effettuate dagli scrittori.
    • Gestisce i contenuti e gli utenti con privilegi speciali.
  2. Interfaccia per i creatori di contenuti (scrittori):
    • Propone nuove voci o modifica quelle esistenti.
    • Le modifiche dovranno essere approvate da un moderatore prima della pubblicazione.
  3. Interfaccia per gli utenti consultatori (front-end):
    • Consente la ricerca e la consultazione delle voci wiki.
    • Non richiede la registrazione per l'accesso ai contenuti.
L’applicazione deve rispettare una serie di requisiti fondamentali.
Deve essere in grado di tenere traccia delle modifiche effettuate, consentendo il ripristino dei contenuti a una versione specifica. Ogni bozza può essere modificata, ma finché una modifica non viene approvata, non è possibile apportarne di nuove alla stessa voce. L’editor deve avere un layout fisso, senza la possibilità di modificarlo. Inoltre, deve essere prevista una form di registrazione per candidarsi a ciascun ruolo, e l’amministratore avrà la possibilità di approvare o rifiutare le candidature tramite email. I campi richiesti per la registrazione includono email, password, classe, anno di nascita, nome e cognome. La password deve avere una lunghezza compresa tra un minimo di 8 e un massimo di 24 caratteri e deve contenere almeno un numero, una lettera e un carattere speciale. Dopo aver effettuato l’accesso tramite l’apposita form, l’utente potrà selezionare il ruolo da utilizzare, qualora ne possieda più di uno, come specificato nei casi d’uso.

Requisiti di Rete e Sicurezza

L'obiettivo è garantire la sicurezza della piattaforma separando il web server pubblico dall'infrastruttura interna che gestisce dati sensibili.
L'architettura è basata sull’ utilizzo di due PC:

  • PC1 - Web Server Apache: situato nella DMZ, che serve i client esterni e gestisce il traffico HTTPS attraverso un tunnel OpenTLS.
  • PC2 - Web Server + Database MySQL + RADIUS: collocato nella rete interna (LAN privata), è accessibile solo da PC1 e non è direttamente esposto a Internet.
Entrambi sono protetti dal firewall di Ubuntu, che controlla il traffico di rete e garantisce che solo connessioni autorizzate possano passare.

PC1 - Web Server Apache (DMZ):
  • Accessibile solo tramite HTTPS su porta 443.
  • Tunnel OpenTLS per garantire una connessione sicura.
  • Firewall di Ubuntu configurato per bloccare tutto il traffico non autorizzato.
  • Utilizzo di Apache.
  • Può comunicare con PC2 sulla LAN privata per interagire con il database o il servizio di autenticazione.
PC2 - Web Server + Database MySQL + RADIUS (LAN privata):
  • NON è esposto direttamente a Internet.
  • Web Server per elaborazione lato server con PHP.
  • Database MySQL per la gestione dei dati.
  • RADIUS per autenticazione sicura degli utenti.
  • Firewall di Ubuntu configurato per accettare solo connessioni da PC1.
  • Blocco di tutte le connessioni non autorizzate dall'esterno.
Firewall per PC1 (DMZ):
  • Accettare connessioni HTTPS sulla porta 443.
  • Permettere connessioni verso PC2 solo sulle porte necessarie (es. 3306 per MySQL, 1812 per RADIUS).
  • Bloccare tutto il traffico in ingresso non autorizzato.
Firewall per PC2 (LAN privata):
  • Accettare connessioni solo da PC1 (nessun accesso diretto da Internet).
  • Permettere connessioni sulla porta 3306 (MySQL) e 1812 (RADIUS) esclusivamente da PC1.
  • Bloccare tutto il traffico proveniente da altre sorgenti.
